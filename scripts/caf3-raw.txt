Chapter 1: Types of Data and Sources
Topic: Types of Data (By Nature)
Q1. A retail store categorizes its customers' payment methods as 'Cash', 'Credit Card', or 'Mobile Wallet'. Since there is no mathematical ranking to these categories, this is an example of: A) Continuous Data B) Ordinal Data C) Nominal Data D) Discrete Data Correct Answer: C Explanation: Nominal data involves descriptive categories that have no inherent mathematical order or ranking
.
Q2. A ride-hailing app asks users to rate their journey as 'Excellent', 'Average', or 'Poor'. What type of data is this? A) Nominal Data B) Ordinal Data C) Discrete Data D) Unstructured Data Correct Answer: B Explanation: Ordinal data contains categories with a meaningful order or sequence, but the exact mathematical distance between the categories cannot be measured
.
Q3. The system dashboard of a logistics company shows exactly 145 active delivery riders in the city. This represents: A) Continuous Data B) Discrete Data C) Qualitative Data D) Semi-structured Data Correct Answer: B Explanation: Discrete data consists of countable, distinct whole numbers that cannot be broken down into fractions (e.g., you cannot have 145.5 riders)
.
Q4. A delivery truck's GPS records its exact distance traveled as 14.65 kilometers. What type of data is this? A) Continuous Data B) Discrete Data C) Nominal Data D) Ordinal Data Correct Answer: A Explanation: Continuous data represents measurable values that can take any value within a range, including decimals and fractions
.

Topic: Categories of Data (By Structure)
Q5. A bank's relational database stores customer transaction histories in strict, predefined rows and columns (e.g., Date, Description, Amount). This is an example of: A) Unstructured Data B) Structured Data C) Semi-structured Data D) Qualitative Data Correct Answer: B Explanation: Structured data is highly organized in predefined formats (like rows and columns in an SQL database), making it easy for systems to process
.
Q6. Customers frequently leave reviews on a company's Facebook page in the form of text paragraphs, emojis, and images. This type of data lacks a predefined format and is known as: A) Structured Data B) Semi-structured Data C) Unstructured Data D) Discrete Data Correct Answer: C Explanation: Unstructured data has no rigid format, making it difficult for computers to process directly (e.g., social media posts, videos, emails)
.
Q7. Which of the following technologies is primarily used by organizations to understand the hidden emotions (e.g., happy, angry) within unstructured customer feedback? A) Natural Language Processing (NLP) B) Sentiment Analysis C) Extensible Markup Language (XML) D) Relational Databases Correct Answer: B Explanation: Sentiment Analysis is a key tech tool used to evaluate the tone and emotion behind unstructured text data
.
Q8. A web application transmits user profile data using JSON (JavaScript Object Notation), which uses tags to organize data but lacks a rigid table format. This is an example of: A) Structured Data B) Unstructured Data C) Semi-structured Data D) Nominal Data Correct Answer: C Explanation: Semi-structured data is not in a rigid table but uses organizational tags or markers (like JSON, XML, or HTML) to separate elements
.

Chapter 2: Data Governance and Management

Q1. The CEO of a multinational bank establishes a vision to migrate 80% of their customer data to a secure cloud environment by 2028 to support analytics-driven decision-making. At which level of data governance is this action taking place? A) Tactical Level B) Operational Level C) Strategic Level D) Execution Level Correct Answer: C Explanation: The strategic level involves top management (like the Board or CEO) setting the vision, goals, and direction for data governance
.
Q2. An IT manager at a retail chain drafts a new password policy requiring all employees to use 12-character passwords and updates the standard operating procedures for data entry. Which level of data governance does this represent? A) Strategic Level B) Tactical Level C) Operational Level D) Compliance Level Correct Answer: B Explanation: The tactical level is where middle management translates the strategic vision into actionable plans, frameworks, and policies
.
Q3. A junior data analyst runs a daily script to identify and flag any missing customer email addresses in the company's CRM system. This daily quality check is an example of data governance at the: A) Strategic Level B) Tactical Level C) Operational Level D) Executive Level Correct Answer: C Explanation: The operational level involves day-to-day staff (like Data Stewards or IT personnel) executing the plans and performing daily data quality checks
.
Q4. What is the primary overarching goal of implementing a data governance framework in an organization? A) To eliminate the need for IT security teams. B) To ensure data is accurate, secure, and compliant with regulations. C) To restrict all employees from accessing internal data. D) To increase the cost of data storage. Correct Answer: B Explanation: Data governance provides the frameworks, policies, and practices needed to ensure data is accurate, secure, and compliant with regulations, transforming it into a trusted asset
.

This data is not meant for general employees but would cause financial risk and internal conflict if leaked. How should this data be classified? This data carries low risk if accidentally seen by an outsider. .
Q8. A university publishes its upcoming fall semester course catalog and marketing brochure on its official website. This data is classified as: A) Public Data B) Internal Data C) Open-Source Data D) Confidential Data Correct Answer: A Explanation: Public data is safe for anyone to view and carries zero risk if accessed by outsiders, such as marketing brochures or website info

This employee resolves discrepancies, merges duplicate entries, and trains store managers on proper data entry practices. Topic: Data Integrity
Q11. A university's database is programmed to reject any student enrollment record where the "Date of Birth" is entered as a date in the future. Which type of data integrity is the database enforcing? A) Physical Integrity B) Entity Integrity C) Referential Integrity D) Logical Integrity Correct Answer: D Explanation: Logical integrity ensures that the data makes logical sense and adheres to defined business rules, such as age not being negative or a birth date not being in the future
.
Q12. When attempting to delete a vendor from the company's ERP system, an accountant receives an error message stating: "Cannot delete vendor. Active purchase orders are still linked to this vendor ID." This control enforces: A) Referential Integrity B) Entity Integrity C) Physical Integrity D) Logical Integrity Correct Answer: A Explanation: Referential integrity maintains valid relationships between tables using Foreign Keys, ensuring that parent records (like a vendor) cannot be deleted if child records (like pending orders) are still linked to them
.
Q13. To ensure that no two citizens are accidentally merged or confused in the national database, the government assigns a unique 13-digit CNIC number to every individual as a Primary Key. This practice maintains: A) Logical Integrity B) Entity Integrity C) Referential Integrity D) Physical Integrity Correct Answer: B Explanation: Entity integrity ensures that every record in a database is unique, typically by using a Primary Key to prevent duplicate entries
.
Q14. A cloud service provider installs temperature sensors, fire suppression systems, and backup generators in its data center to ensure servers remain operational during extreme weather conditions. This protects the database's: A) Referential Integrity B) Entity Integrity C) Logical Integrity D) Physical Integrity Correct Answer: D Explanation: Physical integrity involves protecting the physical hardware from power outages, fires, or damage using backups, UPS, and environmental controls
.

This approach is an example of: A) Fully On-Premises Storage B) Cloud Storage C) Hybrid Storage D) Edge Computing Correct Answer: C Explanation: Hybrid storage is a combination of on-premises and cloud storage, allowing organizations to keep sensitive data on their own servers while utilizing the cloud for less critical, scalable data

.

A) Decision-Making & Action B) Monitoring & Optimization C) Interpretation & Insight Generation D) Data Modelling Correct Answer: D Explanation: Based on the Data Analytics Cycle, the step that immediately follows Data Exploration is Data Modelling
.

Topic: Data Analytics Maturity Model (The 4 Stages)
Q4. According to the Data Analytics Maturity Model, as an organization transitions from Descriptive (hindsight) to Prescriptive (foresight) analytics, which of the following statements is true? A) Both human input and decision complexity decrease. B) Human input in decision-making increases, while information value decreases. C) Human input in decision-making decreases, while the complexity of decisions and information value increases. D) The analytics process becomes simpler and cheaper to execute. .
Q5. A retail company reviews its monthly dashboard to see total sales and website traffic for the previous quarter, answering the question "What happened?". This is an example of: A) Diagnostic Analytics B) Descriptive Analytics C) Predictive Analytics D) Prescriptive Analytics Correct Answer: B Explanation: Descriptive analytics looks at historical data to answer "What happened?" using techniques like data aggregation, visualization, and summarization
.

Which stage of analytics does this represent? A) Descriptive Analytics B) Diagnostic Analytics C) Predictive Analytics D) Prescriptive Analytics Correct Answer: B Explanation: Diagnostic analytics answers "Why did it happen?" by investigating the root causes of a specific issue, such as rising patient readmissions
.
Q7. Which of the following analytical techniques are primarily used in Diagnostic Analytics to identify root causes? A) Drill-down analysis, correlation analysis, and anomaly detection B) Optimization algorithms and decision trees C) Time series analysis and clustering D) Data aggregation and basic pie charts Correct Answer: A Explanation: Diagnostic analytics relies on techniques like drill-down analysis, correlation analysis, and anomaly detection to find out why an event occurred
.
Q8. A supply chain manager builds models to estimate future demand, supplier lead times, and potential stockouts before they occur. This scenario leverages: A) Descriptive Analytics B) Diagnostic Analytics C) Predictive Analytics D) Prescriptive Analytics Correct Answer: C Explanation: Predictive analytics aims to answer "What will happen?" by forecasting future trends, such as anticipating supply chain disruptions
.
Q9. Predictive analytics relies heavily on which of the following foundations? A) Summarizing past sales data manually B) Machine learning models and historical trends C) Creating basic summary statistics D) Detecting outliers from current datasets Correct Answer: B Explanation: Predictive analytics uses machine learning models (like Random Forest or Regression) and historical trends to forecast future outcomes
.
Q10. Which stage of data analytics recommends optimal actions based on predictions to realize maximum business value? A) Descriptive Analytics B) Diagnostic Analytics C) Predictive Analytics D) Prescriptive Analytics Correct Answer: D Explanation: Prescriptive analytics answers "How can I make it happen?" by recommending optimal actions based on predictive data to realize value
.
Q11. An airline system automatically adjusts ticket prices in real-time based on demand forecasts to maximize revenue. This is an application of: A) Descriptive Analytics B) Diagnostic Analytics C) Predictive Analytics D) Prescriptive Analytics Correct Answer: D Explanation: Automatically adjusting and optimizing actions (like ticket pricing) based on demand forecasts leverages prescriptive analytics
.
Q12. If a company wants to utilize optimization algorithms, simulations, and decision trees, they are planning to implement which type of analytics? A) Descriptive Analytics B) Diagnostic Analytics C) Predictive Analytics D) Prescriptive Analytics Correct Answer: D Explanation: Optimization algorithms, simulations, and decision trees are the key techniques used in Prescriptive Analytics
.

Which characteristic of Big Data does this massive scale of data represent? Which Big Data challenge are they facing? .
Q4. A stock exchange processes thousands of high-frequency financial trades in a fraction of a millisecond. The speed at which this data is generated and processed highlights which characteristic of Big Data? A) Veracity B) Variety C) Velocity D) Value Correct Answer: C Explanation: Velocity refers to the high speed and rate at which data is generated, collected, and processed, which is critical in environments like financial markets
.

Chapter 5: Database Management Systems
Topic: DBMS Basics & The ACID Model
Q1. What is the primary function of a Database Management System (DBMS)? A) To create physical network connections between servers B) To store, retrieve, and manage structured and unstructured data efficiently while ensuring security and integrity C) To permanently replace all manual spreadsheet applications D) To format and design front-end user interfaces Correct Answer: B Explanation: A DBMS is a powerful software tool that enables organizations to handle vast amounts of structured and unstructured data with precision, security, and scalability
.
Q2. A customer transfers Rs. 10,000 to another account. The system deducts the money from the sender but crashes before adding it to the receiver. When the system restarts, it completely rolls back the deduction as if it never happened. Which ACID property ensures this "all or nothing" execution? A) Atomicity B) Consistency C) Isolation D) Durability Correct Answer: A Explanation: Atomicity ensures that a transaction is treated as a single, indivisible unit. It guarantees an "all or nothing" execution, meaning if any part of the transaction fails, the entire transaction is rolled back
.
Q3. Two customers try to purchase the last available airline ticket at the exact same millisecond. The DBMS ensures that these concurrent transactions do not interfere with each other. This represents which ACID property? A) Atomicity B) Consistency C) Isolation D) Durability Correct Answer: C Explanation: Isolation ensures that concurrent transactions occur independently without interfering with each other, maintaining data accuracy
.
Q4. Once an online order is confirmed by the DBMS, the changes are permanently saved in the database. Even if a sudden power failure occurs immediately afterward, the order record will not be lost. This is an example of: A) Atomicity B) Consistency C) Isolation D) Durability Correct Answer: D Explanation: Durability guarantees that once a transaction is committed, its effects are permanently saved and recoverable, even in the event of a system crash or power failure
.

Topic: DBMS Architecture (Three-Level Schema)
Q5. In a company's database, the HR department has a view showing employee salaries, while the Sales department has a view showing customer details but cannot see salaries. Which level of the DBMS architecture provides these customized views? A) Internal Level B) Conceptual Level C) External Level D) Physical Level Correct Answer: C Explanation: The External Level (or View Level) is the highest level of DBMS architecture. It provides customized views of the database to different users or departments based on their roles, restricting access to irrelevant data
.
Q6. Which level of the DBMS architecture acts as the global logical view of the entire database, defining entities, attributes, and relationships without worrying about how data is physically stored? A) External Level B) Conceptual Level C) Internal Level D) View Level Correct Answer: B Explanation: The Conceptual Level describes what data is stored in the entire database and the relationships among the data. It defines the logic without specifying physical storage details
.
Q7. The IT department decides to re-index the database and move the data files to a new, faster SSD storage drive. This change does not affect how the users query the data. This represents which level of the DBMS architecture? A) Conceptual Level B) External Level C) Internal Level D) Application Level Correct Answer: C Explanation: The Internal Level (or Physical Level) deals with the physical storage of the database on disks, including the use of file structures, data blocks, and indexing strategies
.

Topic: Data Independence
Q8. The ability to modify the physical storage mechanisms (like changing hard drives or compressing data) without impacting the logical structure or how users access the data is known as: A) Logical Data Independence B) Physical Data Independence C) Entity Integrity D) Referential Integrity Correct Answer: B Explanation: Physical data independence ensures that changes to the internal storage mechanisms (physical storage) do not impact the logical schema or the way data is accessed by users
.
Q9. A database administrator adds a new column, "Customer_Age", to an existing table. This modification to the schema does not break the existing front-end software applications. This is an example of: A) Logical Data Independence B) Physical Data Independence C) Durability D) Atomicity Correct Answer: A Explanation: Logical data independence means that changes to the logical schema (e.g., adding a column) do not affect the external views or the application layer
.

Topic: Entity-Relationship (ER) Model & Cardinality
Q10. In an Entity-Relationship model, an attribute such as 'FullName' that can be further broken down into 'FirstName' and 'LastName' is classified as a: A) Simple Attribute B) Derived Attribute C) Composite Attribute D) Multivalued Attribute Correct Answer: C Explanation: A composite attribute is an attribute that can be broken down into smaller, meaningful sub-attributes, such as breaking a full name into a first name and a last name
.
Q11. A university database shows that one student can enroll in multiple courses, and each course can have multiple students enrolled in it. What type of relationship does this represent? A) One-to-One (1:1) B) One-to-Many (1:N) C) Many-to-Many (M:N) D) Zero-to-Many (0:N) Correct Answer: C Explanation: In a Many-to-Many (M:N) relationship, many entity instances can be associated with many instances of another entity (e.g., students and courses)
.
Q12. What term is used in database design to specify the maximum number of relationship instances in which an entity can participate (such as 1:1, 1:N, or M:N)? A) Atomicity B) Normalization C) Integrity D) Cardinality Correct Answer: D Explanation: Cardinality specifies the maximum number of relationship instances in which an entity can participate, defining whether a relationship is one-to-one, one-to-many, or many-to-many
.

Topic: Types of DBMS
Q13. Which type of Database Management System organizes data in a strict tree-like structure where each child record can have only one parent? A) Relational DBMS B) Network DBMS C) Hierarchical DBMS D) Object-Oriented DBMS Correct Answer: C Explanation: A Hierarchical DBMS organizes data in a tree-like structure where data elements are arranged in parent-child relationships, and each child node can only have one parent
.
Q14. Oracle, MySQL, and Microsoft SQL Server are highly popular systems that organize data into tables containing rows and columns. These are examples of: A) Hierarchical DBMS B) Network DBMS C) Object-Oriented DBMS D) Relational DBMS (RDBMS) Correct Answer: D Explanation: Relational Database Management Systems (RDBMS) store data in table-based structures with rows (records) and columns (attributes). Popular examples include Oracle, MySQL, and SQL Server
.
Q15. Which type of DBMS supports concepts like 'Inheritance' and 'Polymorphism', storing data as entities that contain both the data and the methods used to manipulate it? A) Object-Oriented DBMS (OODBMS) B) Relational DBMS (RDBMS) C) Hierarchical DBMS D) Network DBMS Correct Answer: A Explanation: An Object-Oriented DBMS (OODBMS) stores data as objects containing both data and methods, and it supports advanced programming concepts like inheritance and polymorphism
.
Chapter 6: Database Normalization & Data Warehousing
Topic: Database Normalization (1NF, 2NF, 3NF)

To minimize this data redundancy and ensure data integrity, the database administrator should apply which process? A) Data Extraction B) Database Normalization C) Data Warehousing D) Horizontal Scaling Correct Answer: B Explanation: Database normalization is a systematic process used to organize data in relational databases to minimize redundancy, avoid anomalies, and ensure data integrity
.
Q2. A bank's database table stores a customer's 'Phone Number' attribute. However, several customers have submitted two or three phone numbers, which the data entry clerk has entered into a single cell separated by commas. To achieve First Normal Form (1NF), what must be done? A) Ensure all non-key attributes depend on the entire primary key. B) Eliminate all foreign keys. C) Ensure there are no repeating groups and all attributes are atomic (single-valued). D) Remove transitive dependencies. Correct Answer: C Explanation: A table is in 1NF only if it has no repeating groups and all attributes are atomic, meaning each cell contains only a single, indivisible value
.
Q3. A retail company’s 'Order Details' table uses a composite primary key consisting of both OrderID and ProductID. However, the attribute ProductName depends only on the ProductID and not the OrderID. This violates which normal form? A) First Normal Form (1NF) B) Second Normal Form (2NF) C) Third Normal Form (3NF) D) Boyce-Codd Normal Form (BCNF) Correct Answer: B Explanation: This is a partial dependency. To achieve Second Normal Form (2NF), no non-key attribute should depend on only a part of a composite primary key
.
Q4. In an employee database, the EmployeeID is the primary key. The table includes a DepartmentID column and a DepartmentLocation column. The DepartmentLocation depends on the DepartmentID, which in turn depends on the EmployeeID. To achieve Third Normal Form (3NF), what must the database administrator eliminate? A) Atomic values B) Partial dependencies C) Transitive dependencies D) Composite primary keys Correct Answer: C Explanation: In 3NF, a table must not have transitive dependencies. Non-key attributes (like DepartmentLocation) must not depend on other non-key attributes (like DepartmentID)
.
Q5. When designing a database, an IT team debates whether to fully normalize the data or leave it partially denormalized. What is the primary trade-off of high normalization? A) It increases data redundancy but speeds up all queries. B) It increases storage efficiency and data integrity, but complex queries may run slower due to multiple table joins. C) It entirely eliminates the need for a primary key. D) It allows the database to process unstructured data like videos. Correct Answer: B Explanation: Normalization trade-offs include increased storage efficiency and simplified maintenance, but it can lead to slower query performance because fetching related data requires joining multiple tables
.

Topic: OLTP vs. OLAP & Data Warehousing
Q6. A nationwide supermarket chain processes tens of thousands of customer checkout transactions every hour. The system must quickly insert, update, and delete these short, atomic records in real-time. Which type of system is the supermarket using? A) Online Analytical Processing (OLAP) B) Online Transaction Processing (OLTP) C) Data Warehouse D) Data Mart Correct Answer: B Explanation: Online Transaction Processing (OLTP) systems are optimized for handling a large volume of short, fast, and atomic real-time transactions, such as processing daily sales
.
Q7. The executive board wants to analyze historical sales trends over the past ten years to make strategic business decisions. They query a system that contains integrated, time-variant, and non-volatile data. Which system are they querying? A) An operational ERP system B) A staging area C) A Data Warehouse D) An OLTP database Correct Answer: C Explanation: A data warehouse is subject-oriented, integrated, time-variant, and non-volatile, making it specifically designed for historical data analysis and business intelligence rather than daily transactions
.

Topic: The ETL Process (Extract, Transform, Load)
Q8. During the creation of a data warehouse, raw data is pulled from a CRM system, an ERP system, and several flat files. Before this data is loaded into the main warehouse, it is temporarily stored in an intermediate zone to be scrubbed for duplicates. This zone is called the: A) Presentation Layer B) Staging Area C) Operational Environment D) Data Mart Correct Answer: B Explanation: The Staging Area is used to temporarily store and clean data (e.g., removing duplicates) before loading it into the data warehouse, preventing corruption of the main warehouse during processing
.
Q9. In the ETL process, standardizing date formats across different systems (e.g., converting DD/MM/YYYY to MM/DD/YYYY) and calculating new metrics from raw figures occurs during which phase? A) Extract B) Transform C) Load D) Presentation Correct Answer: B Explanation: The "Transform" phase involves cleaning, standardizing, and enriching the raw data to ensure it is structured and consistent before loading
.
10. To save processing time and bandwidth, a company updates its data warehouse every midnight by only bringing in the new transactions that occurred that specific day, rather than reloading the entire 5-year history. This method is known as: A) Full loading B) Real-time analytical processing C) Incremental loading D) Data purging Correct Answer: C Explanation: Incremental loading in ETL refers to adding only new or updated records to the data warehouse, which is much faster than doing a full data load
.

Topic: Data Warehouse Schemas & Data Marts
Q11. A data architect designs a schema for a new data warehouse. The design features a single central 'Fact Table' containing total sales figures, which is directly linked to denormalized 'Dimension Tables' (like Time, Product, and Customer) radiating outward. This describes a: A) Snowflake Schema B) Star Schema C) Galaxy Schema D) Hierarchical Schema Correct Answer: B Explanation: The Star Schema is the most basic and widely used structure, featuring a central fact table surrounded by denormalized dimension tables, visually resembling a star
.
Q12. To save storage space and ensure stricter data integrity, a database engineer modifies a Star Schema by breaking down the 'Product' dimension table into further sub-tables (e.g., separating 'Product Category' into its own table). This normalized branching structure is a: A) Star Schema B) Galaxy Schema C) Flat File Schema D) Snowflake Schema Correct Answer: D Explanation: The Snowflake Schema is an extension of the Star Schema where the dimension tables are highly normalized into multiple related sub-tables, resembling a snowflake's branching pattern
.
Q13. An enterprise wants to integrate its 'Sales' process and its 'Inventory' process into one analytical model. The database designer creates an architecture with two separate fact tables that share the same dimension tables (like Time and Product). This complex structure is known as a: A) Star Schema B) Snowflake Schema C) Galaxy Schema (Fact Constellation) D) Data Mart Correct Answer: C Explanation: A Galaxy Schema (or Fact Constellation) contains two or more fact tables that share common dimension tables, enabling integrated analysis across multiple business processes
.
Q14. What is the primary trade-off between choosing a Star Schema versus a Snowflake Schema for a data warehouse? A) A Star schema saves storage space, while a Snowflake schema improves query speed. B) A Star schema prioritizes fast query speed at the cost of storage efficiency, while a Snowflake schema saves space but slows down queries due to multiple joins. C) A Star schema is used for real-time transactions, while a Snowflake schema is used for historical data. D) There is no trade-off; they function exactly the same. Correct Answer: B Explanation: Star schemas prioritize query speed over storage efficiency because they are denormalized. Snowflake schemas save storage space through normalization, but the complex joins slow down query performance
.
Q15. The Marketing Department of a large corporation finds the main enterprise data warehouse too massive and slow for their specific daily campaign reporting. To solve this, IT builds them a smaller, customized subset of the warehouse containing only marketing-related data. This subset is called a: A) Data Lake B) Data Mart C) Staging Area D) Relational Database Correct Answer: B Explanation: A Data Mart is a tailored subset of a data warehouse designed to provide faster, focused access to specific business units or departments (like Marketing or HR)
.
